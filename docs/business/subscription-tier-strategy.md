# RouteScout Subscription Tier Strategy

## Purpose

This document defines a recommended free / Plus / Pro model for RouteScout.

It is designed to balance:
- user growth
- product habit formation
- review generation
- product data collection
- cost control
- subscription conversion

## Core Principle

RouteScout should not lock all meaningful product use behind a paywall.

The free tier should be good enough for people to:
- run with the app
- track runs
- set goals
- feel motivated by it
- build attachment to the product

The paid tiers should unlock:
- more route scouting
- more planning depth
- more saved value
- more personalization

This is especially important because:
- route generation is likely more expensive than basic run tracking
- run usage gives the product valuable data
- habit formation is critical for conversion

## Recommended Tier Model

### Free

Free should answer:
- can I try RouteScout for real?
- can I run with it?
- can I start building a habit?

Recommended free features:
- account creation
- basic run tracking
- basic route-follow / navigation
- basic goal setting
- one active Scout Spec at a time
- one active route at a time
- limited route scouting volume
- limited saved history
- basic post-run summary

Recommended free limits:
- low monthly route-generation cap
- limited saved routes
- limited saved Scout Specs
- no advanced route comparison
- no advanced explanation layer
- no route collections or trip packs

The free tier should feel:
- real
- motivating
- useful

It should not feel like:
- a fake teaser

## Why Free Should Include Run Use

This is an important product and economics decision.

If RouteScout only becomes compelling once someone actually uses a route on a run, then it is good
strategy to let people try that real-world experience for free in a limited way.

Benefits:
- stronger adoption
- better reviews
- more product habit
- more data
- more believable upgrade pressure

The cost-control mechanism should primarily be:
- generation limits

not:
- blocking all run-day use

## Plus

Plus should be the main subscription product.

It should answer:
- I like RouteScout, how do I use it seriously?

Recommended Plus features:
- higher or unlimited route scouting volume
- multiple Scout Specs
- full saved-route collection
- saved plan/spec variants
- route comparison
- route explanations
- richer route history
- trip route packs
- cross-surface continuity between web and iPhone
- better planning depth and reuse

Plus should feel like:
- RouteScout becomes a real planning system

This is the main paid tier for:
- frequent runners
- travel runners
- people who start using the planner regularly

## Pro

Pro should not just be "more Plus."

It should be a meaningfully different product layer.

Recommended Pro direction:
- RouteScout learns the runner

That can include:
- learned pace and run-duration patterns
- learned preferences for hills, pavement, quietness, or route complexity
- adaptive route suggestions
- simple-input route recommendations like:
  - "30 minutes from here"
  - "easy run from hotel"
  - "hill workout near work"
- explanation layer:
  - why this route fits the runner
  - what history or preferences informed the suggestion
- coaching-style nudges
- training-aware route proposals

Pro should feel like:
- RouteScout knows you

This is a later-tier vision, not necessarily an MVP launch requirement.

## Clean Positioning By Tier

### Free

Run with RouteScout.

### Plus

Plan with RouteScout seriously.

### Pro

RouteScout knows you.

## User Progression

The ideal user journey is:

1. discover RouteScout
2. try it without much friction
3. run with it for real
4. build trust and habit
5. start wanting more planning power
6. upgrade to Plus
7. generate enough history and preference data that Pro becomes meaningful later

This is important because RouteScout is not just a tool people read about.
It is a tool they need to feel in real use.

### Stage 1: Curiosity

The user is drawn in by:
- a route card
- a social post
- a search result
- a travel-use case
- a recommendation from another runner

At this stage, the product needs to feel easy to try.

### Stage 2: Real Use

The user:
- creates a route
- follows it on a run
- tracks the run
- sees that RouteScout is not just marketing

This is why free should include real run use in a limited way.

### Stage 3: Habit

The user returns and starts to think:
- I want more route options
- I want to save more things
- I want this to fit my routine better

This is where Plus becomes compelling.

### Stage 4: Deeper Reliance

The user:
- uses multiple Scout Specs
- builds saved route collections
- compares options
- starts to expect RouteScout to help with planning, not just route display

This is the core paid behavior.

### Stage 5: Personal Intelligence

After enough history exists, the user may want:
- RouteScout to understand their preferences
- route suggestions from simple prompts
- lightweight coaching and training-aware route proposals

This is where Pro becomes believable.

## Why The Tier Boundaries Make Sense

### Why Free Should Be Real

If free feels fake, users will not trust the product enough to care.

Free should therefore include:
- actual run use
- actual goal engagement
- enough route scouting to understand the value

That improves:
- activation
- retention
- reviews
- word of mouth

### Why Plus Should Focus On Planning Depth

Plus should not be just "remove a few limits."

It should feel like the product becomes more serious:
- more route scouting
- more saved value
- better comparison
- better reuse
- better planning across trips and routine runs

That makes the subscription feel like an upgrade in capability, not just a tax.

### Why Pro Should Wait For Real Intelligence

Pro should not exist just to create a pricing ladder.

It should exist only when RouteScout genuinely:
- learns from the runner
- improves its recommendations based on history
- creates a new layer of value beyond standard planning

That protects the product from weak tiering and helps Pro feel premium instead of arbitrary.

## Recommended Launch Strategy

For launch, the cleanest approach is:
- ship `Free`
- ship `Plus`
- define `Pro`, but do not require it to launch unless the personalized-intelligence layer is real

Why:
- two tiers are easier to explain
- you avoid weak artificial tier separation
- you can still sell a clear premium product

## Recommended Free / Plus Boundary

The strongest practical boundary is:

Free:
- basic use
- basic habit formation
- capped scouting
- single-spec simplicity

Plus:
- deeper route scouting
- route collections
- multiple specs
- reuse and comparison
- serious planning value

This keeps the upgrade tied to:
- planning power
- saved value
- repeat usefulness

## Pricing Logic

This tier model supports:
- a free acquisition layer
- a mainstream paid tier
- a future high-value premium tier

Suggested pricing logic:
- Free
- Plus monthly and annual
- Pro later when the personalized intelligence layer is genuinely differentiated

## Bottom Line

The best RouteScout tier model is probably:

- Free = enough real value to build habit and trust
- Plus = enough planning depth to justify a subscription
- Pro = enough intelligence and personalization to justify a premium tier later

That model supports:
- user growth
- review growth
- better product data
- healthier conversion pressure
- stronger long-term product strategy
