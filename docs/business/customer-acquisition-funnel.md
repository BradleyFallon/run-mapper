# RouteScout Customer Acquisition Funnel

## Purpose

This document maps the intended customer-acquisition path for RouteScout from first awareness to
subscription.

It is designed to answer:
- where users come from
- how they move from curiosity to account creation
- how they move from the web planner into the iPhone app
- where subscription conversion should happen
- which channels can feed the funnel

## Core Funnel Idea

The product pitch is:

"RouteScout scouts out the run for you."

The funnel should let users feel that value before asking for the biggest commitment.

That means:
- awareness first
- curiosity second
- route-planning investment third
- account creation after emotional investment
- mobile app handoff after route value is visible
- subscription request at the point where the route becomes useful in real life

## Funnel Tree

### 1. Awareness Sources

Primary acquisition sources:
- founder social posts
- promoted social posts
- running communities
- travel-running crossover communities
- Reddit posts used selectively
- Google search / SEO
- App Store search
- referrals and shared Route Cards
- direct outreach / personal network

These sources should all point into one of two entry surfaces:
- the web planner
- the iPhone app listing

For MVP, the web planner should be the main landing surface.

## 2. Web Entry

The user lands on the RouteScout web planner from:
- a social post
- a paid promotion
- a Google search result
- a shared route or referral link

The page should immediately answer:
- what RouteScout is
- why it is different
- how to try it quickly

The message should be:
- not "browse routes"
- but "tell RouteScout what kind of run you want and let it scout the best option"

## 3. Planning Engagement

The user begins planning by entering things like:
- location
- distance or time
- route preferences
- terrain / surface priorities
- confidence or discovery preferences

This is an important psychological stage.

By this point, the user is no longer passively reading.
They are investing time and intent into a route they want.

That is why account creation should happen after meaningful planning engagement, not before it.

## 4. Registration Gate

Recommended behavior:
- let the user begin the planning flow without creating an account
- let them invest effort into the plan
- show a partial preview or route teaser
- require registration to reveal, save, or unlock the full result

This is the right conversion point because:
- the user is now curious
- the route feels personalized
- the product has already demonstrated value

The account gate should feel like:
- "Create an account to reveal and save your route"

It should not feel like:
- "Sign up before we will show you anything"

## 5. Route Reveal

After registration, the user can:
- view the route
- inspect route details
- understand why it was selected
- save it to their account

At this point, the user should feel:
- this route is relevant
- this route feels custom
- this is better than trying to figure it out manually

## 6. Mobile App Handoff

The next step should be:
- prompt the user to open the route in the iPhone app

This can be presented as:
- "Use this route in the app"
- "Open on iPhone for full route use"
- "Take this run with you"

The point of the handoff is:
- web does planning
- iPhone does real route use

## 7. Subscription Gate

The subscription gate should happen at the point where the route becomes fully useful:
- navigation
- route use on the phone
- route tracking / run-time features

Recommended free behavior:
- users can scout a route
- users can create an account
- users can preview and save a limited route result

Recommended paid behavior:
- navigation / real route use in the iPhone app
- ongoing route access and premium planning depth
- richer saved-route value

This keeps the paywall attached to real value, not just abstract access.

## 8. Trial Conversion

The expected conversion path is:
- user sees the route
- user wants to try actually using it
- user starts a subscription trial
- user tries the route in real life
- user decides whether the route planning was worth paying for

This is the right moment for the trial because the user is close to real-world use.

## Important App Store Constraint

For iPhone subscriptions, the standard App Store model is auto-renewable subscriptions.

That means:
- introductory free trials and introductory pricing are designed to renew unless the user cancels

This is the normal Apple pattern.

So the product should not promise a special "free trial that does not auto-renew" as the standard
subscription model.

Instead, the non-scammy approach is:
- be explicit that the trial will renew unless canceled
- show the renewal terms clearly
- remind users before renewal if possible
- make cancellation expectations clear

Official Apple references:
- auto-renewable subscriptions and introductory offers: https://developer.apple.com/app-store/subscriptions/
- introductory offers in App Store Connect: https://developer.apple.com/help/app-store-connect/manage-subscriptions/set-up-introductory-offers-for-auto-renewable-subscriptions
- offer codes that renew to standard price after the offer duration: https://developer.apple.com/help/app-store-connect/manage-subscriptions/set-up-subscription-offer-codes/

## Pricing Strategy In The Funnel

The goal early on is:
- maximize learning
- maximize product usage
- maximize proof of demand

That means pricing should be easy to accept.

Recommended structure:
- monthly subscription around `$4.99`
- simple introductory offer
- strong beta / early-adopter offer

Possible founder-offer logic:
- a limited "founding runner" annual plan at `$10` or similar

This kind of very low annual offer should be treated as:
- a launch or beta-era special
- not necessarily the permanent standard annual price

## Channel Branches

### Social / Promoted Social

Path:
- post
- click to planner
- plan route
- register
- view route
- open app
- trial
- subscription

Good for:
- emotion
- curiosity
- travel-running stories
- visual Route Card sharing

### Google Search / SEO

Path:
- search query
- landing page or planner page
- plan route
- register
- app handoff
- subscription

Good for:
- high-intent users
- travel searches
- "best running route in [city]" type discovery

### App Store Search

Path:
- find app in App Store
- install
- onboard
- sign in / account create
- scout route
- trial
- subscription

Good for:
- high-intent mobile-first users

### Referral / Shared Route

Path:
- receive shared route or Route Card
- open RouteScout
- become curious
- try planning their own route
- register
- app handoff
- subscription

Good for:
- trust
- social proof
- lower-friction entry

## Growth Flywheel

This is not a pure network-effect product in the strict sense.

It is better described as a growth flywheel:
- more users
- more routes planned
- more shared routes and Route Cards
- more feedback
- better product quality
- more social proof
- more referrals and discovery

That is still valuable.
It just should be described accurately.

## What Success Looks Like

A healthy early funnel would look like:
- strong clickthrough from social and search
- decent planner start rate
- meaningful completion rate through planning
- solid registration conversion after route investment
- strong mobile handoff rate
- healthy trial start rate
- enough post-trial conversion to prove real willingness to pay

## Recommended Founder Position

The customer-acquisition plan should be described like this:

"We want to get people curious enough to scout a route, let them invest real intent into planning
it, then require account creation to unlock and save the result. Once they see the value, we move
them into the iPhone app where the route becomes truly usable. That is where the subscription trial
should happen. The idea is to ask for commitment only after the product has done enough to feel
valuable."
