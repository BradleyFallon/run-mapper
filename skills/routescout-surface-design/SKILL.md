## Design Direction

Maintain the current product direction:
- web app = planning-first
- iOS app = full product and navigation surface

Maintain the current visual direction:
- dark theme by default
- restrained color
- high contrast
- premium, calm, athletic feel

## Surface Design Rules

- Start with structure before visual polish.
- Prefer workspace-style flows for planning tasks.
- Keep `category`, `plan`, and `route` distinctions obvious in the UI.
- Keep web planning docs desktop-first.
- Keep iPhone design focused on route use, saved items, and navigation readiness.

## Avoid

- Jumping into high-fidelity UI before structure is clear
- Mixing web-planner behavior with in-run mobile navigation behavior
- Reintroducing `mode` as the primary UX object
